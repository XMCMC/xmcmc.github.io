Main page

Contributing, support, and project norms.

Contributing

  1. Open an issue describing the bug, feature, or design question.
  2. Fork the repository and create a focused branch.
  3. Add tests and documentation for user-facing changes.
  4. Open a pull request with a short explanation and reproducible example.

Support

Use GitHub issues for reproducible bugs and feature requests. For design discussions, create a discussion thread or issue with the design label.

Roadmap

  • Finalize first sampler API.
  • Add examples and diagnostics.
  • Publish package installation instructions.
  • Add citation and license information.
