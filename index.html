XMCMC is an R package for writing fast, compiled Bayesian MCMC models with user-defined parameter update blocks. It is useful when you know the structure of your model and want to tell the sampler exactly how each parameter should be updated, including optional deterministic caches for expensive repeated calculations.

For more details, see the XMCMC website and the source repository.

Why XMCMC?

XMCMC is designed for models where generic MCMC systems spend time recomputing work that the model author knows how to avoid. A model can define scalar, grouped, and discrete update blocks, and those blocks are compiled into fast model-specific code.

Install

Install the development version from GitHub:

install.packages("remotes")
remotes::install_github("XMCMC/XMCMC")

XMCMC compiles model-specific C code and a Rust sampler, so you need a working C compiler and Rust cargo on your system path.

First example: Gaussian regression

The model below simulates a regression problem with an x matrix, estimates the regression coefficients, and estimates the residual standard deviation through logsigma.

library(XMCMC)

set.seed(1)

n <- 1000L
p <- 5L

x <- matrix(rnorm(n * p), nrow = n, ncol = p)
x <- cbind(1, x)        # first column is the intercept
p <- ncol(x)

beta_true <- c(0.5, 1.0, -0.75, 0.5, 0.0, -0.25)
sigma_true <- 0.75

y <- as.numeric(x %*% beta_true + rnorm(n, 0, sigma_true))

data <- list(n = n, p = p, x = x, y = y)
model_src <- '
param beta(p)
param logsigma(1)

func y_lpdf() {
  double sigma = exp(logsigma(1));

  for (i = 1:n) {
    double mu = 0.0;

    for (j = 1:p) {
      mu += beta(j) * x(i,j);
    }

    y(i) ~ dnorm(mu, sigma);
  }
}

block beta(j) {
  beta(j) ~ dnorm(0, 10);
  y_lpdf();
}

block logsigma(1) {
  logsigma(1) ~ dnorm(0, 2);
  y_lpdf();
}
'

fit <- XMCMC(
  model = model_src,
  data = data,
  samples = 2000,
  burnin = 1000,
  out = "chain_regression.bin"
)

draws <- read_XMCMC("chain_regression.bin")
head(draws)

Cache optimization

In the plain model, each scalar coefficient proposal rebuilds the full linear predictor. For large p, that is wasteful because changing beta(j) only changes mu(i) by one rank-one update.

model_src_cached <- '
param beta(p)
param logsigma(1)

func y_lpdf() {
  double sigma = exp(logsigma(1));

  for (i = 1:n) {
    y(i) ~ dnorm(mu(i), sigma);
  }
}

block beta(j) {
  beta(j) ~ dnorm(0, 10);
  y_lpdf();
} cache mu(n) {
  for (i = 1:n) {
    mu(i) = 0.0;

    for (k = 1:p) {
      mu(i) += beta(k) * x(i,k);
    }
  }
} update mu(n) {
  for (i = 1:n) {
    mu(i) += (proposal(beta(j)) - current(beta(j))) * x(i,j);
  }
}

block logsigma(1) {
  logsigma(1) ~ dnorm(0, 2);
  y_lpdf();
}
'

The cached model is mathematically the same model as the plain version. The difference is computational: mu(n) is initialized once and then maintained incrementally as one coefficient changes at a time.
